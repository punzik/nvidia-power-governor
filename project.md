# Описание проекта

Демон, регулирующий мощность видеокарт Nvidia в соответствии с температурой чипа.

Цель: не давать видеокартам перегреваться под интенсивной нагрузкой, плавно регулируя ограничение максимальной мощности через утилиту `nvidia-smi`.

## Реализация

Программа должна быть написана на чистом Си без применения сторонних библиотек, кроме libc. Должна статически линковаться, чтобы не зависеть от дистрибутива и окружения.

Программа работает как long-running foreground-процесс (без daemonization).

### Конфигурационный файл

Формат — INI-подобный. Секции: `[global]` и `[gpu.N]`, где N — порядковый номер GPU. Каждая директива в формате `ключ=значение`. Поддерживаются комментарии (`#` или `;` в начале строки).

Пример:

```ini
[global]
poll_interval=1000
avg_samples=5
power_step=1
hysteresis=3

[gpu.0]
max_temp=80
max_power=300
min_power=50

[gpu.1]
max_temp=75
max_power=250
min_power=50
```

Конфигурацией задаются общие параметры:

- Частота опроса в миллисекундах.
- Количество отсчётов для усреднения показаний температуры.
- Шаг регулировки мощности в ваттах.
- Гистерезис температуры в градусах Цельсия.

Для каждой видеокарты задаются:

- Максимальная температура (°C).
- Максимальная мощность (W).
- Минимальная мощность (W).

GPU идентифицируются по порядковому номеру (0, 1, …).

### Аргументы командной строки

Парсинг через `getopt_long`:

- `-c` / `--config <path>` — путь к файлу конфига (обязательный)
- `-h` / `--help` — вывод справки и выход
- `--restore-max` — установить мощность всех GPU на `max_power` из конфига, затем выйти
- `--restore-min` — установить мощность всех GPU на `min_power` из конфига, затем выйти
- `--restore-factory` — запросить default power limit у каждого GPU через `nvidia-smi` и установить его, затем выйти

### Взаимодействие с nvidia-smi

- Установка мощности: `nvidia-smi -i <id> -pl <W>` (мощность в ваттах)
- Persistence mode включать не нужно
- Подсчёт GPU: `nvidia-smi -L`, подсчёт строк
- Чтение температуры: `nvidia-smi -i <id> -q -d TEMPERATURE`, парсинг `GPU Current Temp`
- Чтение factory default: `nvidia-smi -i <id> -q -d POWER`, парсинг `Default Power Limit` (в mW, конвертировать в W)

### Алгоритм регулирования

Для каждого GPU на каждой итерации опроса:

1. Считать текущую температуру, записать в кольцевой буфер
2. Вычислить среднюю температуру по всем образцам в буфере
3. Применить правило:
   - Если `avg_temp >= max_temp` → снизить мощность на `power_step` (но не ниже `min_power`)
   - Если `avg_temp <= max_temp - hysteresis` → повысить мощность на `power_step` (но не выше `max_power`)
   - В промежуточной зоне — не менять

### Логирование

Весь вывод — в stdout. Программа не передаёт структурированных данных, только логи.

### Обработка ошибок

- При любой ошибке (неудачный вызов `nvidia-smi`, ошибка парсинга конфига и т.д.) — вывести сообщение об ошибке и завершиться
- При старте проверять, что количество GPU на системе совпадает с количеством GPU в конфиге; при несовпадении — ошибка и выход

### Сборка

Простой Makefile. Статическая линковка.

## Тестирование

На локальном компьютере установлены программы `gpu-burn` и `nvidia-smi`.
