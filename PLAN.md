# План реализации nvidia-power-governor

## Структура проекта

```
main.c        — парсинг аргументов, main-цикл, restore-логика
config.h      — структуры конфигурации
config.c      — парсер INI-конфига
gpu.h         — типы и прототипы GPU-операций
gpu.c         — чтение температуры, установка мощности, запрос default power
regulate.h    — функция регулирования мощности
regulate.c    — алгоритм регулирования мощности
test.c        — unit-тесты
Makefile      — сборка
example.conf  — пример конфигурации
tests/        — тестовые конфиги
```

Один `main.c` + три модуля (`config`, `gpu`, `regulate`) — достаточно для KISS-подхода.

---

## Шаг 1. Определить структуры данных

**config.h**

- `struct gpu_config`: `id` (int), `max_temp` (int °C), `max_power` (int W), `min_power` (int W), `power_step` (W)
- `struct global_config`: `poll_interval` (ms), `avg_samples` (int), `hysteresis` (°C)
- `struct config`: `global`, массив `gpu_configs`, `gpu_count`

**gpu.h**

- `struct gpu_state`: `id`, `current_power` (W), кольцевой буфер температур (размер = `avg_samples`), индекс буфера

## Шаг 2. Реализовать парсер конфига (`config.c`)

INI-подобный формат:

```ini
[global]
poll_interval=1000
avg_samples=5
hysteresis=3

[gpu.0]
max_temp=80
max_power=300
min_power=50
power_step=1

[gpu.1]
max_temp=75
max_power=250
min_power=50
power_step=1
```

Функции:

- `config_load(path)` — парсит файл, возвращает `struct config *` (или `NULL` при ошибке). В продакшене обёртка `config_load_or_die()` выводит ошибку и вызывает `exit(1)`.
  - Пропускает пустые строки и строки-комментарии (`;` или `#` в начале)
  - Секция `[global]` → заполняет `global_config`
  - Секция `[gpu.N]` → заполняет элемент массива `gpu_configs`
  - Каждая директива: `key=value`, trim пробелов, `strtol` для парсинга значений
  - Возвращает `NULL` при любой ошибке (файл не найден, некорректный формат, невалидное значение)
- `config_free(cfg)` — освобождает память

## Шаг 3. Реализовать GPU-операции (`gpu.c`)

Все взаимодействия с `nvidia-smi` через `popen()` + парсинг вывода.

Функции:

- `gpu_count()` — запускает `nvidia-smi -L`, подсчитывает строки. Возвращает количество GPU.
- `gpu_read_temp(id)` — `nvidia-smi -i <id> -q -d TEMPERATURE`, парсит строку `GPU Current Temp`. Возвращает int °C.
- `gpu_set_power(id, power_w)` — `nvidia-smi -i <id> -pl <power_w>`. Проверяет возвращаемый код.
- `gpu_default_power(id)` — `nvidia-smi -i <id> -q -d POWER`, парсит `Default Power Limit` (в mW), конвертирует в W. Возвращает int W.
- `gpu_state_init()` — выделяет и инициализирует массив `gpu_state` (current_power = max_power из конфига, буферы температур = 0)

При любой ошибке `popen` / парсинга / ненулевом rc — `fprintf(stderr)` + `exit(1)`.

## Шаг 4. Реализовать алгоритм регулирования (`regulate.c`)

Вынести логику регулирования в отдельную чистую функцию для тестируемости.

**regulate.h**

- `regulate_compute(avg_temp, gpu_cfg, hysteresis, current_power) → new_power`
  - Чистая функция: принимает среднюю температуру, конфиг GPU, гистерезис и текущую мощность
  - Возвращает новую мощность (W) или ту же, если не изменилась
  - Логика:
    - Если `avg_temp >= max_temp` → `current_power - power_step` (но не ниже `min_power`)
    - Если `avg_temp <= max_temp - hysteresis` → `current_power + power_step` (но не выше `max_power`)
    - Иначе — вернуть `current_power` без изменений

## Шаг 5. Реализовать main-цикл (`main.c`)

После загрузки конфига и валидации GPU count:

1. Инициализировать `gpu_state` для каждого GPU
2. Установить начальную мощность каждого GPU на `max_power` (из конфига)
3. Цикл:
   - Для каждого GPU: считать температуру → записать в кольцевой буфер
   - Для каждого GPU: вычислить среднюю температуру по всем образцам в буфере
   - Для каждого GPU: вызвать `regulate_compute()` и применить результат
   - Для каждого GPU, у которого изменилась мощность → `gpu_set_power()`
   - `usleep(poll_interval * 1000)`

## Шаг 6. Реализовать парсинг аргументов и restore-режимы (`main.c`)

Аргументы командной строки через `getopt_long`:

- `-c` / `--config <path>` — путь к файлу конфига (обязательный)
- `-h` / `--help` — вывод справки и выход
- `--restore-max` — установить мощность всех GPU на `max_power` из конфига, затем выйти
- `--restore-min` — установить мощность всех GPU на `min_power` из конфига, затем выйти
- `--restore-factory` — запросить default power limit у каждого GPU и установить его, затем выйти

Логика `main()`:

1. Распарсить аргументы
2. Если `--help` → вывести usage → `exit(0)`
3. Если путь к конфиг не указан → ошибка → `exit(1)`
4. Загрузить конфиг
5. Проверить, что `gpu_count()` == `config.gpu_count` → если нет → ошибка → `exit(1)`
6. Если `--restore-*` → выполнить restore → `exit(0)`
7. Иначе → перейти к main-циклу (Шаг 4)

## Шаг 7. Написать Makefile

- Компилятор: `cc` (или `gcc` из окружения)
- Флаги: `-Wall -Wextra -O2 -std=c11`
- Статическая линковка: `-static`
- Цель `all`: линкует `main.o config.o gpu.o regulate.o` → `nvidia-power-governor`
- Цель `clean`: удаляет `.o` и бинарник
- Цель `install` (опционально): копирует бинарник в `/usr/local/bin`
- Цель `test`: собирает `test.o config.o` → `test_runner`, запускает его. Не требует `gpu.c` и реальных GPU.

## Шаг 9. Написать unit-тесты (`test.c`)

Простой self-contained тест-фреймворк (assert + printf, без сторонних библиотек). Запуск через `make test`.

### Тесты парсера конфига

- Валидный конфиг (два GPU) — все поля распаршены корректно
- Валидный конфиг (один GPU)
- Пустой файл — возвращает `NULL`
- Отсутствующий файл — возвращает `NULL`
- Файл только с комментариями и пустыми строками — возвращает `NULL`
- Некорректное числовое значение — возвращает `NULL`
- Неизвестный ключ — возвращает `NULL`
- Пропущенная секция `[global]` — возвращает `NULL`
- Пропущенная секция GPU — возвращает `NULL`
- Пробелы вокруг `=` и значений — обрабатываются корректно

### Тесты алгоритма регулирования

- Температура выше `max_temp` → мощность снижена на `power_step`
- Температура выше `max_temp`, но уже достигнут `min_power` → мощность не изменилась
- Температура ниже `max_temp - hysteresis` → мощность повышена на `power_step`
- Температура ниже `max_temp - hysteresis`, но уже достигнут `max_power` → мощность не изменилась
- Температура в промежуточной зоне → мощность не изменилась
- Температура ровно на границе `max_temp` → снижение
- Температура ровно на границе `max_temp - hysteresis` → повышение

## Шаг 10. Ручное тестирование

1. Собрать проект: `make`
2. Проверить `--help`
3. Проверить работу с некорректным конфигом (отсутствующий файл, пустой файл, несовпадение GPU count)
4. Запустить с `example.conf`, убедиться, что температура читается корректно
5. Запустить `gpu-burn`, убедиться, что мощность снижается при перегреве
6. Проверить `--restore-max`, `--restore-min`, `--restore-factory`
